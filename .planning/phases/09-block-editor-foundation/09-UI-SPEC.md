# Phase 9 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope:** This phase fixes core Blockly block editor functionality. The visual design is primarily driven by Blockly's built-in styling, with custom CSS for isolation and theming. shadcn components are used for editor chrome (toolbars, panels, dialogs) but NOT for the block workspace itself.

## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing, Blockly block internal padding |

| 2xl | 48px | Major section breaks |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` | Workspace background, main editor surface |
| Secondary (30%) | `hsl(240 4.8% 95.9%)` | Panels, toolbars, palette background |
| Accent (10%) | `hsl(240 5.9% 10%)` | Zoom controls, reset view button, active block outline |
| Destructive | `hsl(0 84.2% 60.2%)` | Trashcan, delete block confirmation |

**Dark mode equivalents:**
- Dominant: `hsl(240 10% 3.9%)`
- Secondary: `hsl(240 3.7% 15.9%)`
- Accent: `hsl(0 0% 98%)`

**Accent reserved for:**
- Zoom controls (+/- buttons, fit-to-screen)
- Reset view button
- Active block selection outline
- Valid drop target highlight
- Connection snap indicator

**Blockly block colors:** Use standard Blockly color scheme for block categories:
- Robot blocks: 230 (purple)
- Logic blocks: 210 (blue)
- Loop blocks: 120 (green)
- Math blocks: 230 (purple)
- Text blocks: 160 (teal)
- Variables: 330 (pink)

**Source:** shadcn zinc preset (pre-populated from components.json)

## Visual Feedback States

**Block interaction states:**

| State | Visual Treatment | Timing |
|-------|------------------|--------|
| Hover (idle) | Block shadow: 2px offset, 0.2 opacity | Immediate |
| Hover (drag) | Block shadow: 8px offset, 0.3 opacity, scale 1.02 | Immediate |
| Selected | Outline: 2px accent color, rounded 4px | Immediate |
| Valid drop | Connection highlight: 3px accent stroke, rounded 2px | < 100ms |
| Invalid drop | Ghost block returns to origin, no snap | < 300ms animation |

**Workspace navigation:**
- Pan: Scroll wheel (smooth, no acceleration)
- Zoom: Ctrl+scroll, centered on cursor
- Zoom range: 30% - 300% (Blockly defaults)
- Zoom speed: 1.2x per scroll tick

**Source:** RESEARCH.md workspace configuration + Blockly defaults

## Performance Notes

**Block editor performance targets (from PERF-01, PERF-02):**
- Initial workspace load: < 3 seconds
- Block selection response: < 100ms
- Drag feedback: < 16ms (60fps)

**Optimization strategies:**
- Use `event.isUiEvent` filtering to prevent code generation lag
- ResizeObserver for responsive workspace sizing
- CSS transforms for zoom (no layout thrashing)

5. **BLOCK-05 (Visual feedback)**: Requires CSS z-index hierarchy and `.blocklyDraggable` styling

**Verification:**

- Test with 50+ blocks to verify performance doesn't degrade
